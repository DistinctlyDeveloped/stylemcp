Documentation

StyleMCP validates and rewrites AI-generated text to match your brand voice. Use it via REST API, MCP, or CLI.

Base URL: https://stylemcp.com

Quick Start

Validate text against brand rules with a single API call:

cURL 
curl -X POST https://stylemcp.com/api/validate \
  -H "Content-Type: application/json" \
  -d '{"text": "Click here to learn more!"}'

Response:

JSON 
{
  "valid": false,
  "score": 65,
  "violations": [
    {
      "rule": "no-click-here",
      "severity": "error",
      "message": "Avoid 'click here' - describe the destination instead",
      "suggestion": "Learn more about our features"
    }
  ]
}

Authentication

If API key authentication is enabled, include your key in the X-Api-Key header:

cURL 
curl -X POST https://stylemcp.com/api/validate \
  -H "Content-Type: application/json" \
  -H "X-Api-Key: your-api-key" \
  -d '{"text": "Your text here"}'

Validate Text

POST /api/validate

Check if text matches your brand rules.

Request Body

FieldTypeDescription
textstringRequired. The text to validate
packstringStyle pack to use (default: "saas")
contextobjectAdditional context (e.g., content type)

Response

FieldTypeDescription
validbooleanWhether text passes all rules
scorenumberQuality score (0-100)
violationsarrayList of rule violations
suggestionsarrayImprovement suggestions

Rewrite Text

POST /api/rewrite

Automatically rewrite text to match your brand voice.

cURL 
curl -X POST https://stylemcp.com/api/rewrite \
  -H "Content-Type: application/json" \
  -d '{
    "text": "Please click here to utilize our product!",
    "mode": "normal"
  }'

Rewrite Modes

ModeDescription
minimalOnly fix rule violations
normalFix rule violations + improve clarity
aggressiveComplete rewrite in brand voice

Batch Validation

POST /api/validate/batch

Validate multiple texts in a single request:

cURL 
curl -X POST https://stylemcp.com/api/validate/batch \
  -H "Content-Type: application/json" \
  -d '{
    "items": [
      {"id": "headline", "text": "Revolutionary AI Solution!"},
      {"id": "cta", "text": "Click here to get started"},
      {"id": "body", "text": "We help teams work better."}
    ]
  }'

MCP Integration

StyleMCP supports the Model Context Protocol for direct integration with AI agents like Claude.

SSE Endpoint

GET /api/mcp/sse

Tool Calls

POST /api/mcp/call

Available MCP tools:

  • validate_text - Validate text against brand rules
  • rewrite_to_style - Rewrite text to match brand voice
  • get_voice_rules - Get voice and tone guidelines
  • get_copy_patterns - Get approved copy patterns
  • get_cta_rules - Get CTA guidelines
  • list_packs - List available style packs

GitHub Actions

Run brand validation in CI:

.github/workflows/brand-check.yml 
name: Brand Check
on: [pull_request]

jobs:
  validate:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Check copy
        run: |
          npx stylemcp validate src/copy/*.json \
            --min-score 80 \
            --format github

CLI

Install and use the CLI:

Terminal 
# Install
npm install -g stylemcp

# Validate text
stylemcp validate "Click here to learn more"

# Validate file
stylemcp validate src/copy/homepage.json

# Rewrite text
stylemcp rewrite "Please utilize our product" --mode aggressive

Pack Structure

A style pack is a directory containing YAML/JSON files:

Directory Structure 
packs/
  my-brand/
    manifest.yaml      # Pack metadata and config
    voice.yaml         # Tone, vocabulary, forbidden words
    copy_patterns.yaml # Reusable copy templates
    cta_rules.yaml     # Button and CTA guidelines
    tokens.json        # Design tokens (optional)

manifest.yaml

YAML 
name: my-brand
version: "1.0.0"
description: "Brand voice for My Company"

files:
  voice: voice.yaml
  copyPatterns: copy_patterns.yaml
  ctaRules: cta_rules.yaml

config:
  strictMode: false
  minScore: 70

voice.yaml

YAML 
tone:
  summary: "Friendly, clear, and helpful"
  attributes:
    - name: friendly
      weight: 0.8

vocabulary:
  rules:
    - preferred: "use"
      avoid: ["utilize", "leverage"]

  forbidden:
    - "synergy"
    - "game-changing"

doNot:
  - pattern: "click here"
    reason: "Poor accessibility"
    suggestion: "Describe the destination"

Custom Packs

To create your own style pack:

  1. Copy the default pack: cp -r packs/saas packs/my-brand
  2. Edit the YAML files to match your brand guidelines
  3. Update manifest.yaml with your pack name
  4. Use it via API: {"pack": "my-brand"}

Tip: Start with the saas pack and modify it. Most of the rules are good defaults for professional B2B copy.
